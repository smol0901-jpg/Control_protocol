DevCleaner Pro – автономное Windows‑приложение, которое в реальном времени отслеживает dev‑процессы, предлагает завершить «тяжёлые», очищает кэши разработки и генерирует PDF‑отчёты. Использует машинное обучение для предсказания безопасности завершения процессов и бережёт ресурсы ПК благодаря адаптивному регулятору нагрузки.

![Логотип](https://via.placeholder.com/800x200?text=DevCleaner+Pro+Logo)

## 🚀 Ключевые возможности

- **Умный мониторинг** – находит процессы Node.js, Python, PowerShell, Java, компиляторов и др., анализирует нагрузку (CPU/ОЗУ) и предлагает завершить.
- **Автоматический помощник** – каждые 2 минуты открывает окно со списком тяжёлых процессов, позволяя завершить их в один клик.
- **ML‑анализ** – модель машинного обучения (Random Forest) предсказывает, какие процессы можно безопасно завершить. Постоянно дообучается на ваших действиях.
- **Очистка кэшей** – удаляет временные файлы npm, pip, Yarn, Gradle, Maven, Electron, VS Code и др., освобождая место на диске.
- **PDF‑отчёты** – генерирует красивые отчёты с графиками, статистикой ML и рекомендациями.
- **HUD с графиками** – отдельное окно с графиками загрузки CPU и памяти в реальном времени (энергосберегающий режим при сворачивании).
- **HTTP API** – позволяет управлять программой удалённо (получить список процессов, завершить, очистить кэш, создать отчёт).
- **Лёгкий режим** – временно отключает автоматическое обучение ML, экономя ресурсы процессора.
- **Адаптивная нагрузка** – при высокой загрузке CPU программа автоматически снижает свою активность.
- **Полное логирование** – все события записываются в структурированные логи с автоматической ротацией (не более 100 МБ).

## 🖥️ Системные требования

- **ОС**: Windows 10 / 11 (64‑бит)
- **Процессор**: Intel Celeron N4120 1.1 ГГц или аналогичный
- **ОЗУ**: 8 ГБ (рекомендуется)
- **Место на диске**: ~500 МБ для установки
- **Права**: требуются права администратора для завершения некоторых процессов и установки

## 📦 Установка

### Способ 1: Инсталлятор (рекомендуется)
1. Скачайте последнюю версию установщика из [Releases](https://github.com/yourname/devcleaner-pro/releases).

## 🧭 Использование

После запуска в системном трее появится зелёная иконка **DevCleaner Pro**. Нажмите правой кнопкой для доступа к меню:

| Пункт меню | Описание |
|------------|----------|
| 👤 Войти | Автоматическая авторизация (admin/admin123!) для доступа к отчётам и API. |
| 📊 Статистика | Показывает сводку: убито процессов, освобождено памяти, очищено кэша. |
| 🖥️ Главное окно | Полный интерфейс с вкладками «Процессы», «Статистика», «Управление», «Логи». |
| 🔍 Сканировать сейчас | Мгновенно находит тяжёлые процессы и открывает окно с возможностью завершить. |
| 📄 Создать отчет | Генерирует PDF‑отчёт и открывает папку с ним. |
| 📺 HUD с графиками | Запускает отдельное окно с графиками загрузки CPU и памяти. |
| ⚡ Лёгкий режим | Включает/выключает режим экономии ресурсов (без автообучения ML). |
| 🌐 API Сервер | Включает HTTP API на порту 5050. |
| ❌ Выход | Закрывает программу. |

### HTTP API (после включения)

Базовый URL: `http://127.0.0.1:5050/api`

| Метод | Эндпоинт | Описание |
|-------|----------|----------|
| POST | `/login` | Получить токен (тело: `{"username":"admin","password":"admin123!"}`). |
| GET | `/processes` | Список dev‑процессов. |
| POST | `/kill/<pid>` | Завершить процесс по PID. |
| POST | `/kill_all` | Завершить все безопасные процессы. |
| POST | `/clean_cache` | Очистить все кэши разработки. |
| GET | `/stats` | Статистика программы и системы. |
| GET | `/report?type=daily` | Сгенерировать отчёт (daily, weekly, monthly). |

## ⚙️ Конфигурация

Файл `C:\Users\<Имя>\.devcleaner_pro\config.json` позволяет настроить:

- `alert_memory_mb` / `alert_cpu_percent` – пороги для уведомлений о тяжёлых процессах.
- `auto_kill` – автоматически завершать процессы, которые ML‑модель считает безопасными.
- `scan_interval_seconds` – интервал сканирования процессов (по умолчанию 30 сек).
- `auto_report_interval_hours` – как часто создавать отчёты (по умолчанию 24 часа).
- `api_enabled` – запускать ли HTTP API при старте программы.

## 📁 Логи

Все события записываются в `C:\Users\<Имя>\.devcleaner_pro\error_logs\`.  
Логи автоматически ротируются: **1000 строк на файл**, общий объём не превышает **100 МБ**.  
Старые файлы удаляются при превышении лимита.
